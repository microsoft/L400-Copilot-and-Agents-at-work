## Task 02: Apply Lifecycle Management practices

### 01: Configure deployment environments
Set up Dev, UAT, and PRD as managed deployment environments, including names, IDs, and descriptions, to establish the lifecycle foundation.

1. On the **Deployment Pipeline Configuration**, select the ellipses **(...)**, then select the **play icon**.

1. On the left side menu, under **Pipeline Setup**, select **Environments**.

1. On the top action menu, select **New** to proceed with the pipeline configuration.

1. Fill in the following information:
   - Name: `Zava Retail Store - Dev`
   - Owner: `Deployment Pipeline Administrators`
   - Environment Type: `Development Environment`
   - Environment ID: `d1159265-2104-e7c3-8168-220f7286f30e`
   - Description: `Environment for managing application deployment stages.`

1. Select **Save & Close**.

1. Repeat the same steps for `Zava Retail Store - UAT`:
   - Name: `Zava Retail Store - UAT`
   - Owner: `Deployment Pipeline Administrators`
   - Environment Type: `Target Environment`
   - Environment ID: `f026f3fc-7cb6-e37e-8a6e-a34db192d31c`
   - Description: `Environment for managing application deployment stages.`

1. Repeat the same steps for `Zava Retail Store - PRD`:
   - Name: `Zava Retail Store - PRD`
   - Owner: `Deployment Pipeline Administrators`
   - Environment Type: `Target Environment`
   - Environment ID: `e26bd40d-88a2-eb35-bd11-b98445f9c511`
   - Description: `Environment for managing application deployment stages.`

### 02: Build and enforce the deployment pipeline
Create a pipeline linking Dev → UAT → PRD and define deployment flow rules that strictly enforce the promotion sequence, ensuring deployments move from Dev to UAT first and only UAT deployments can be promoted to production.

1. On the left side menu, under **Pipeline Setup**, select **Pipelines**.

1. On the top action menu, select **+ New**.

1. Fill in the deployment pipeline fields by using this information:

   - **Name**: `Dev → UAT → PRD`
   - **Description**: `Automated deployment pipeline and version movement for Dev, UAT, and PRD.`
   - **Owner**: `Deployment Pipeline Administrators`

1. Select **Save** and scroll down to the **Linked Development Environments** section.

1. Under **Linked Development Environments**, select **Add Existing Development Environment**, then select `Zava Retail Store - Dev` and **Add**.

1. Scroll further to the **Deployment Stages (Deployment Pipeline)** section and select **New Deployment Stage**.

1. Fill in the deployment stage fields with this information:

      - **Name**: `Dev → UAT Deployment Rules`
      - **Owner**: `Deployment Pipeline Administrators`
      - **Description**: `The deployment process must first progress from Dev to UAT.`      

1. Leave **Previous Deployment Stage** empty as it is Dev.

1. Type in the **Target Deployment Environment ID** field: 

    ```
    Zava Retail Store - UAT
    ```

1. Select **Save & Close**.

1. Repeat the same steps and fill in the following:
   - **Name**: `UAT → PRD Deployment Rules`
   - **Owner**: `Deployment Pipeline Administrators`
   - **Description**: `Only deployments originating from UAT are permitted to be promoted to production.`

1. Select the search icon next to **Previous Deployment Stage**, and set it to `Dev → UAT Deployment Rules`.

1. Select the search icon next to **Target Deployment Environment ID** and set it to `Zava Retail Store - PRD`.

1. Select **Save & Close**.

1. Under **Pipeline Setup**, select **Pipelines** and notice the new pipeline was created.

1. Let's verify that the pipeline is created by going to the second tab in the browser tab bar - **Power Apps - Apps Tab**. 


1. In the left side menu, select **Solution**, then **ZavaRetailsSolution** solution to open it.

1. In the left side menu, select the **Pipelines** icon and notice the **Dev → UAT → PRD** pipeline we just created.
